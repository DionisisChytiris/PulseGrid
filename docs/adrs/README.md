# Architecture Decision Records (ADR)

This folder contains Architecture Decision Records for PulseGrid.

An ADR documents important technical decisions.

Goals:

* Explain why decisions were made.
* Record alternatives considered.
* Capture tradeoffs.
* Preserve engineering context.

## When to Create an ADR

Create an ADR whenever:

* A major architectural choice is made.
* Multiple alternatives exist.
* A decision has long-term consequences.
* The team spends significant time discussing a solution.

Examples:

* Choosing state management.
* Choosing database technology.
* Choosing navigation framework.
* Selecting monetization strategy.

## ADR Template

Status:

Accepted | Proposed | Deprecated | Superseded

Context:

What problem are we solving?

Decision:

What was chosen?

Rationale:

Why was this option selected?

Consequences:

Positive and negative outcomes.

Alternatives:

What was considered and rejected?
